1. Kiedy Git wykona fast-forward, a kiedy powstaje merge commit?

od momentu utworzenia nowej galezi nie zmienilo sie 
-Merge commit nastepuje wtedy kiedy w galezi docelowej pojawily sie zmiany kiedy 
pracowalismy na innej galezi.Git musi stworzyc nowy commit laczacy obie linii

-Rebase ma czytelna historie i wszystko przenosi na koniec aktualnego stanu main
